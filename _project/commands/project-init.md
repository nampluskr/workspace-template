# 커스텀 명령어: `project-init` / `프로젝트 초기화`

프로젝트 시작 시 또는 진행 중 `project-init 실행` 또는 `@project-init.md 실행` 으로 호출한다.
사용자와 대화형으로 프로젝트 정보를 수집하여 `_project/PROJECT.md` → `_project/PROJECT-TODO.md` → `README.md` 순으로 작성하거나 갱신한다.

## 실행 모드 판단

명령어 실행 시 먼저 `_project/PROJECT.md` 존재 여부를 확인한다.

- **신규 모드**: `_project/PROJECT.md` 가 없거나 플레이스홀더({...}) 상태인 경우 → 전체 항목을 새로 수집하여 작성
- **업데이트 모드**: `_project/PROJECT.md` 에 실제 내용이 있는 경우 → 현재 값을 보여주며 변경할 항목만 수집하여 갱신

## 질문 절차

아래 순서대로 사용자에게 질문한다.
업데이트 모드에서는 각 질문 앞에 `[현재: ...]` 형태로 기존 값을 먼저 보여준다.
사용자가 빈 입력(Enter)을 하면 기존 값을 유지한다.

### Step 1. 기본 정보

```
1. 프로젝트명을 입력하세요.
2. 프로젝트를 한 줄로 설명해 주세요.
3. Subject code를 선택하세요.
   MATH, PHYS, NA, ML, CV, NLP, CS, DEV, MISC
   (복수 선택 가능, 쉼표로 구분)
```

### Step 2. 프로젝트 내용

```
4. 이 프로젝트의 목적은 무엇인가요? (왜 이 작업을 하는가)
5. 배경이나 문제 상황을 설명해 주세요.
6. 수행할 작업의 범위를 항목별로 입력해 주세요. (여러 줄 가능, 완료 시 빈 줄 입력)
7. 제약 사항이 있으면 입력해 주세요. (환경, 도구, 시간 등. 없으면 Enter)
```

### Step 3. 진행 단계

앞서 수집한 목적, 배경, 범위를 바탕으로 AI CLI 가 먼저 Stage - Phase 수준의 초안을 작성하여 제시한다.

```
수집한 내용을 바탕으로 진행 단계 초안을 작성했습니다.

Stage 1 {Stage명}
  Phase 1.1 {Phase명}
  Phase 1.2 {Phase명}
Stage 2 {Stage명}
  Phase 2.1 {Phase명}
  ...

수정할 사항이 있으면 말씀해 주세요. (없으면 "확인" 입력)
```

사용자 피드백을 반영하여 Stage/Phase 를 수정하고, "확인" 입력 시 다음 단계로 넘어간다.
수정이 반복될 경우 매번 전체 구조를 다시 출력하여 현재 상태를 보여준다.

진행 단계가 확정되면 AI CLI 가 각 Phase 에 맞는 Task 를 직접 작성하여 `_project/PROJECT-TODO.md` 초안을 생성한다.

## 확인 및 수정

모든 항목 수집 후 요약을 보여주고 사용자에게 확인을 요청한다.

```
수집한 내용을 요약합니다.
프로젝트명: ...
설명: ...
Subject code: ...
목적: ...
배경: ...
범위: ...
제약 사항: ...
진행 단계:
  Stage 1 ...
    Phase 1.1 ...
      Task: ...
이 내용으로 파일을 작성합니다. 수정할 항목이 있으면 항목 번호 또는 이름을 말씀해 주세요.
확인이면 "확인" 또는 "진행"을 입력하세요.
```

수정 요청 시 해당 항목만 재질문 후 재확인한다.

## 폴더 구조 확인 및 생성

모든 항목 확인 완료 후, 파일 작성 전에 폴더 구조를 확정한다.
확정된 폴더 구조는 README.md 의 디렉토리 구조 섹션에 반영된다.

### Step 1. 현재 폴더 구조 출력

워크스페이스 템플릿의 기본 폴더 구조를 아래 형식으로 보여준다.

```
현재 워크스페이스 폴더 구조입니다.

project-root/
├── _project/         # 워크스페이스 운영 문서
├── _assets/            # 읽기 전용 입력 보관소
├── scripts/            # 실행 스크립트 및 Python 파일
├── outputs/            # 프로젝트 산출물
└── docs/               # Jupyter Book v2 최종 산출물
    └── contents/
```

### Step 2. 추가 폴더 생성 여부 확인

```
추가로 생성할 폴더가 있으면 폴더명을 입력하세요.
(여러 개는 쉼표로 구분, 없으면 Enter)
```

- 사용자가 폴더명을 입력하면 루트에 해당 폴더를 생성한다.
- 빈 입력(Enter)이면 폴더 생성을 건너뛴다.
- 생성 후 결과를 출력한다.

```
폴더 생성 완료: {폴더명1}/, {폴더명2}/
```

폴더 구조가 확정되면 파일 작성을 시작한다.

## 파일 작성 규칙

### PROJECT.md

`_project/rules/markdown-style.md` 규칙을 준수하여 작성한다.

```markdown
# {프로젝트명}

{한 줄 설명}

> 생성일시: YYMMDD-HHMMSS
> 수정일시: YYMMDD-HHMMSS
> 주제: {Subject Name}

**목차**

1. [목적](#1-목적)
2. [배경](#2-배경)
3. [범위](#3-범위)
4. [제약 사항](#4-제약-사항)
5. [진행 단계](#5-진행-단계)

## 1. 목적

{수집한 목적}

## 2. 배경

{수집한 배경}

## 3. 범위

{수집한 범위 항목 불릿}

## 4. 제약 사항

{수집한 제약 사항 불릿}

## 5. 진행 단계

### 5.1. Stage 1 {Stage명}

- Phase 1.1 {Phase명}
- Phase 1.2 {Phase명}

### 5.2. Stage 2 {Stage명}

- Phase 2.1 {Phase명}
```

### PROJECT-TODO.md

```markdown
# 작업 목록

진행 현황 대시보드입니다. Stage - Phase - Task 단위로 체크박스를 관리합니다.
Stage 및 Phase 구성은 PROJECT.md 의 단계 섹션을 기준으로 합니다.

## Stage 1 {Stage명}

### Phase 1.1 {Phase명}

- [ ] Task 1
- [ ] Task 2

### Phase 1.2 {Phase명}

- [ ] Task 1

## Stage 2 {Stage명}

### Phase 2.1 {Phase명}

- [ ] Task 1
```

**업데이트 모드 주의사항**:
- 완료된 체크박스(`- [x]`)는 절대 변경하지 않는다.
- 기존 Phase에 Task를 추가할 때는 미완료 항목 뒤에 이어 붙인다.
- 새로운 Stage/Phase는 기존 내용 뒤에 추가한다.

### README.md

신규 모드에서는 프로젝트명, 설명, 태그, 개요, 확정된 디렉토리 구조 섹션을 반영한다.
업데이트 모드에서는 프로젝트명, 설명, 태그만 갱신한다. 나머지 섹션은 유지한다.

## 완료 보고

파일 작성 완료 후 아래 형식으로 보고한다.

```
완료되었습니다.

작성된 파일:
- _project/PROJECT.md: 목적 / 배경 / 범위 / 제약 / {N}개 Stage
- _project/PROJECT-TODO.md: {N}개 Stage, {M}개 Phase, {K}개 Task
- README.md: 프로젝트명 / 설명 / 태그 갱신

생성된 폴더: {폴더명1}/, {폴더명2}/   ← 추가 폴더가 없으면 이 항목 생략

다음 단계: PROJECT-TODO.md 의 Stage 1 Phase 1.1 부터 작업을 시작하세요.
```
