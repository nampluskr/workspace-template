## 1. 운영 원칙

이 문서는 100개 이상의 단위 프로젝트를 수행할 때 전체 현황을 빠르게 파악하기 위한 단일 관리 파일이다. 초기 단계에서는 별도의 색인, 백로그, 주제별 통계 파일을 만들지 않고 이 문서에서 통합 관리한다.

프로젝트 항목은 사용자가 제목과 간단한 개요를 제시하면 AI CLI 가 주제, 상태, 예상 산출물, 다음 행동을 정리하여 이 문서에 반영한다. 실제 프로젝트 폴더는 항목이 `ready` 또는 `active` 상태가 된 뒤 생성한다.

프로젝트의 subject code 는 `_workspace/rules/subjects.md` 를 기준으로 한다.

## 2. 상태 정의

프로젝트 상태는 다음 값을 사용한다.

| 상태 | 의미 |
| --- | --- |
| `idea` | 제목과 간단한 아이디어만 있는 상태 |
| `draft` | 목적과 범위가 초안 수준으로 정리된 상태 |
| `ready` | 바로 프로젝트 폴더를 생성하고 착수할 수 있는 상태 |
| `active` | 현재 진행 중인 상태 |
| `paused` | 일시 중단된 상태 |
| `completed` | 산출물 정리와 문서화가 완료된 상태 |
| `archived` | 보관 대상이며 추가 진행 계획이 없는 상태 |
| `dropped` | 진행하지 않기로 결정한 상태 |

프로젝트 ID 는 `P0001` 형식을 사용한다. 새 항목은 가장 마지막 ID 다음 번호를 부여한다.

## 5. 사용자 입력 양식

사용자는 아래와 같이 제목과 간단한 개요만 작성하여 프로젝트 항목 추가를 요청한다.

```text
프로젝트 항목 추가:
- 제목: {프로젝트 제목}
- 개요: {프로젝트의 목적, 배경, 만들고 싶은 산출물에 대한 간단한 설명}
```

AI CLI 는 입력 내용을 바탕으로 다음 작업을 수행한다.

- `_workspace/rules/subjects.md` 기준으로 subject code 를 판정한다.
- 제목을 간결한 프로젝트명으로 정리한다.
- 개요를 1문장으로 정제한다.
- 예상 산출물을 `문서`, `코드`, `데이터`, `이미지`, `Jupyter Book`, `운영 문서` 중 적절한 값으로 정리한다.
- 다음 행동을 짧은 실행 단위로 작성한다.
- `프로젝트 항목` 표에 새 행을 추가한다.
- `주제별 프로젝트 현황` 표의 수치를 갱신한다.
