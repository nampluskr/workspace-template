# 커스텀 명령어: `/commit-message`

사용자가 "커밋 메시지", "commit-message", "/commit-message" 라고 요청하면 아래 절차를 실행합니다.
커밋은 사용자가 직접 수행합니다. AI CLI 는 메시지만 제안합니다.

---

## 실행 절차

1. `git status` 로 변경 파일 목록을 확인합니다.
2. `git diff HEAD --stat` 로 변경 규모를 파악합니다.
3. 변경 규모가 크면 주요 파일만 `git diff HEAD -- {파일}` 로 선택 확인합니다.
4. 커밋 메시지를 제안합니다. 변경 규모가 클 경우 단일·분리 두 가지를 함께 제안합니다.

---

## 메시지 형식

```
{type}({scope}): {subject}
```

| 항목 | 설명 |
| --- | --- |
| `type` | 변경 유형 (아래 표 참조) |
| `scope` | 변경 범위 (폴더명, 파일명, 기능명 등) |
| `subject` | 변경 내용 요약 (명령형, 50자 이내) |

### type 목록

| type | 사용 시점 |
| --- | --- |
| `feat` | 새 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 추가/수정 |
| `chore` | 설정 또는 운영 변경 |
| `refactor` | 코드 리팩토링 |
| `style` | 포맷, 공백 등 기능 변경 없는 수정 |
| `test` | 테스트 추가/수정 |

---

## 제안 형식

### 단일 커밋인 경우

```
{type}({scope}): {subject}
```

### 분리 커밋이 필요한 경우

변경 규모가 크거나 성격이 다른 변경이 혼재할 때 함께 제안합니다.

```
커밋 1: {type}({scope}): {subject}
커밋 2: {type}({scope}): {subject}
```
