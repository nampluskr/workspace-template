# 커스텀 명령어: `session-end`

세션 종료 전 `session-end 실행` 또는 `@session-end.md 실행` 으로 호출합니다.
TASKS.md 업데이트, workspace-history.md 갱신, 세션 핸드오프 문서 작성을 순서대로 수행합니다.

---

## 실행 절차

### Step 1. 작업 결과 확인

1. `git status` 로 이번 세션에서 변경된 파일 목록을 확인합니다.
2. 완료된 Task 를 사용자에게 확인합니다.

### Step 2. TASKS.md 업데이트

1. 사용자가 확인한 완료 항목을 `[ ]` → `[x]` 로 변경합니다.
2. 완료되지 않은 항목은 그대로 유지합니다.

### Step 3. workspace-history.md 업데이트

`_workspace/docs/workspace-history.md` 에 이번 세션 내역을 추가합니다.

1. **변경 이력 요약 테이블**에 한 줄 추가합니다.

   ```
   | YYYY-MM-DD | {N}차 | {주요 변경 내용 한 줄 요약} |
   ```

2. **세션별 상세 내역** 섹션에 신규 항목을 추가합니다.

   ```markdown
   ### 2.{N} {N}차 세션 — {세션 주제}

   > 참조: `_workspace/sessions/{파일명}`

   **배경**

   {이번 세션 작업의 배경 및 목적}

   **신규 생성 파일**

   | 파일 | 내용 |
   | --- | --- |
   | `...` | ... |

   **수정 파일**

   | 파일 | 변경 내용 |
   | --- | --- |
   | `...` | ... |

   **주요 결정사항**

   | 항목 | 결정 내용 |
   | --- | --- |
   | ... | ... |
   ```

   신규 생성 파일 또는 수정 파일이 없는 경우 해당 항목은 생략합니다.

### Step 4. session-handoff 문서 작성

`session-handoff` 절차를 실행하여 `_workspace/sessions/` 에 핸드오프 문서를 저장합니다.

### Step 5. 종료 브리핑 출력

아래 형식으로 브리핑을 출력합니다.

```
== 세션 종료 브리핑 ==

[ 완료된 작업 ]
- [x] Task 1
- [x] Task 2

[ 미완료 / 미결 ]
- [ ] Task 3
- 미결 사항: ...          ← 없으면 이 항목 생략

[ 저장된 파일 ]
- TASKS.md 업데이트 완료
- workspace-history.md 업데이트 완료
- 핸드오프: _workspace/sessions/{파일명}

[ 다음 세션 시작 지시문 ]
"session-start 실행 후 {다음 Task} 를 이어서 진행해 주세요."
```
