# 커스텀 명령어: `session-handoff`

세션 종료 전 `session-handoff 실행` 또는 `@session-handoff.md 실행` 으로 호출한다.
이 파일은 명령어의 문서 템플릿 및 작성 원칙 참고용이다.

---

## 저장 위치

```
_workspace/sessions/YYMMDD-HHMMSS_session-handoff.md
```

파일 저장 전에 반드시 `date +%y%m%d-%H%M%S` 를 실행하여 실제 현재 시각을 확인하고 그 값을 파일명에 사용한다. 플레이스홀더(`HHMMSS`, `000000` 등)를 그대로 쓰는 것을 금지한다.

---

## 작성 원칙

- 단순 요약이 아니라, 다른 세션에서 이 문서만 읽어도 즉시 작업을 이어받을 수 있도록 작성한다.
- 사용자가 명시적으로 확정한 내용과 아직 논의 중인 내용을 명확히 구분한다.
- 앞으로 할 작업은 다른 세션에서 바로 지시할 수 있는 수준으로 구체화한다.
- 기술 용어, 파일명, 폴더명, 결정사항은 정확하게 그대로 기록한다.
- 사용자의 의도와 맥락이 드러나도록 작성한다 (표면적 요청 + 숨겨진 목적).

---

## 기본 템플릿

```markdown
# [주제] 세션 핸드오프

> 작성일: YYYY-MM-DD
> 세션 목적: (한 줄로)
> 이전 핸드오프: {파일명, 없으면 이 줄 생략}

---

## 1. 세션 핵심 요약

(이 세션에서 논의된 이슈를 모두 불릿 형식으로 작성. 확정/미결 구분 없이 언급된 모든 항목 포함)

---

## 2. 사용자 요청 및 의도

(사용자가 요청한 것과 그 배경 목적을 구분하여 기록)

| 요청 내용 | 배경 목적 |
| --- | --- |
| ... | ... |

---

## 3. 확정된 결정사항

(이 세션에서 사용자가 명시적으로 확정한 내용만 기록)

| 항목 | 확정 내용 | 비고 |
| --- | --- | --- |
| ... | ... | ... |

---

## 4. AI 핵심 제안 및 분석

(AI 가 제안하고 사용자가 수용한 내용을 불릿으로 작성. 비교가 필요한 경우 테이블 추가)

- **제안 제목**
  - 근거 및 배경
  - 사용자 수용 이유
  - (필요시) 비교 테이블

---

## 5. 미결 사항

(논의했지만 아직 결정되지 않은 항목)

| # | 항목 | 현재 상태 | 결정 필요 내용 |
| --- | --- | --- | --- |
| 1 | ... | 검토중 | ... |

---

## 6. 다음 작업 목록

(이 세션 이후 해야 할 작업을 우선순위 순으로)

| 우선순위 | 작업 | 관련 폴더/파일 | 사용할 도구 |
| --- | --- | --- | --- |
| 1 | ... | ... | ... |

---

## 7. 다음 세션에서 이어받기

(새 세션에서 바로 붙여넣고 시작할 수 있는 지시문)

### 새 세션 시작 지시문:

[이 문서를 붙여넣은 후 아래 지시문으로 시작]

"아래는 이전 세션의 컨텍스트입니다.
 이 내용을 기반으로 [다음 작업]을 진행해 주세요.

 특히 다음 항목을 우선적으로 다뤄주세요:
 1. ...
 2. ..."

---

## 8. 참고 자료 및 링크

(세션 중 언급된 파일, 문서, URL, 도구 등)

| 항목 | 위치/경로 | 용도 |
| --- | --- | --- |
| ... | ... | ... |
```

### 작성 시 추가 지침

- 섹션 3 (확정된 결정사항) 은 가장 중요하므로 빠짐없이 작성한다.
- 섹션 6 (다음 작업) 은 AI CLI 에게 바로 지시할 수 있는 수준으로 구체적으로 작성한다 (추상적 표현 금지).
- 섹션 7 (새 세션 시작 지시문) 은 복사-붙여넣기 즉시 사용 가능하게 작성한다.
- 전문 용어, 파일명, 경로는 정확하게 그대로 기록한다.
- 마크다운 코드블록, 표, 목록을 적극 활용하여 구조화한다.

---

## 변형 버전

세션 성격에 따라 아래 변형 버전을 사용한다.

### 짧은 버전 (빠른 정리용)

세션이 짧거나 단순할 때 사용한다.

```markdown
이 세션을 간단히 정리해 주세요. 아래 4가지만 마크다운으로 작성:

1. **핵심 요약** (3줄 이내)
2. **확정된 내용** (표 형식)
3. **미결 사항** (목록)
4. **다음 할 일** (번호 목록, 구체적으로)
```

### 기술 작업 특화 버전

코딩, 설계, 아키텍처 논의 후 사용한다.

```markdown
이 기술 세션을 정리해 주세요. 아래 구조로 마크다운 작성:

1. **작업 컨텍스트** (환경, 언어, 프레임워크, 제약조건)
2. **확정된 설계/구현** (코드, 구조, 명령어 포함)
3. **미해결 기술 이슈** (오류, 미구현, 검토 필요)
4. **다음 구현 단계** (AI CLI 에 바로 지시할 수 있게)
5. **참고 파일/경로** (실제 경로 그대로)

코드 블록과 정확한 파일명/경로를 반드시 포함해 주세요.
```

### 학습 내용 정리 특화 버전

딥러닝 / 수치해석 / 알고리즘 학습 세션 후 사용한다.

```markdown
이 학습 세션을 정리해 주세요. 아래 구조로 마크다운 작성:

1. **학습 주제** (핵심 개념 3~5개)
2. **이해한 내용** (본인 언어로 재서술된 핵심)
3. **수식 / 코드 핵심** (LaTeX 수식, 코드 스니펫 포함)
4. **아직 불명확한 부분** (추가 학습 필요)
5. **관련 주제 연결** (이 개념과 연결되는 다른 주제)
```
