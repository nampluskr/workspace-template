## 4. AI 핵심 제안 및 분석

- **세션 파일명 오류 원인**
  - `260605-HHMMSS`: AI가 핸드오프 작성 시 `date` 명령을 실행하지 않고 플레이스홀더를 그대로 파일명으로 사용
  - `260606-000000`: AI가 시각을 확인하지 않고 임의로 자정(`000000`)을 기입
  - 공통 원인: `session-end.md` / `session-handoff.md` 에 실제 시각 확인 지침이 없었음

- **SSOT 방식 불채택 근거**
  - CLAUDE.md / AGENTS.md 는 AI CLI 가 세션 시작 시 자동으로 컨텍스트에 로드하는 파일
  - 내용을 외부 파일로 분리하면 AI가 해당 파일을 능동적으로 Read 해야 하므로 자동 로드 보장 불가
  - 관리 파일이 2개뿐이라 수동 복사 부담이 크지 않음
